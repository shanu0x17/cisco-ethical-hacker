# Lab 02 – Using OSINT Tools

## Objective

Explore commonly used Open Source Intelligence (OSINT) tools and understand how publicly available information can be used during the reconnaissance phase of a penetration test.

---

## Environment

- Operating System: Kali Linux
- Platform: Cisco Ethical Hacker Virtual Lab
- Tools Used:
  - OSINT Framework
  - WhatsMyName
  - SpiderFoot
  - Recon-ng

---

## Activities Performed

- Explored the OSINT Framework to understand different categories of publicly available intelligence resources.
- Performed username-based investigations using WhatsMyName.
- Executed a passive reconnaissance scan using SpiderFoot.
- Explored Recon-ng modules and workspaces.
- Investigated publicly available information without directly interacting with the target systems.

---

## Tools Overview

| Tool | Purpose |
|------|---------|
| OSINT Framework | Directory of OSINT resources categorized by investigation type |
| WhatsMyName | Searches for usernames across multiple online platforms |
| SpiderFoot | Automates OSINT collection from various public sources |
| Recon-ng | Framework for reconnaissance and information gathering |

---

## Key Observations

- OSINT relies entirely on publicly available information.
- A single username can reveal accounts across multiple online services.
- SpiderFoot automates data collection from numerous intelligence sources.
- Recon-ng provides a modular framework for structured reconnaissance.
- Passive reconnaissance minimizes direct interaction with the target while collecting useful intelligence.

---

## Skills Practiced

- Open Source Intelligence (OSINT)
- Passive Reconnaissance
- Username Enumeration
- Digital Footprint Analysis
- Reconnaissance Framework Navigation

---

## Key Learnings

- Publicly available information can reveal valuable intelligence about individuals and organizations.
- Multiple OSINT tools can be combined to build a comprehensive digital footprint.
- Effective reconnaissance improves the planning and efficiency of later penetration testing phases.
- Passive information gathering reduces the likelihood of detection during the early stages of an assessment.


---

## Personal Reflection

This lab provided practical exposure to several widely used OSINT tools. I learned how publicly available information can be collected, correlated, and analyzed during the reconnaissance phase without directly interacting with the target. The exercise highlighted the importance of passive intelligence gathering before performing active security assessments.
